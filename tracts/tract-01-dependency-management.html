Implementation Strategy

Critical Path (Sequential Dependencies)

These tickets MUST be completed in sequence. Each level blocks the next.

Level 1: Foundation (Start Here)

  • BOB-001 Security & Access Control - Gate for all user interactions
  • BOB-002 Bundle Configuration Dropdown - Required for bundle selection throughout app
  • BOB-005 Document Category Structure - Foundation for document organization
  • BOB-008 Activity Log Database Schema - Must exist before any job tracking
  • BOB-007 BytePro API Integration (Byte 25.2) - External dependency, critical blocker for all processing

Level 2: Logic Engines (Build After Level 1)

  • BOB-006 Hide Not Applicable Docs Logic Engine - Needed by grid display (BOB-034)
  • BOB-009 PDF Destination Mapping - Required for PDF delivery (BOB-054, BOB-055, BOB-056)
  • BOB-010 Manual Refresh Pattern - Needed by grid operations

Level 3: Loan Identification (Build After Level 2)

  • BOB-021 Manual Entry - Foundation for all identification methods
  • BOB-025 View Loan Population - Gate to grid view (depends on validation logic)
  • BOB-027 Maximum Population Size Validation - Must be enforced before grid loads

Level 4: Grid Foundation (Build After Level 3)

  • BOB-031 Grid Structure - Base for all grid features
  • BOB-032 Filter Tabs - Core navigation within grid
  • BOB-035 Multi-Select Checkboxes - Required by BOB-038 (Build Selected)

Level 5: Background Jobs (Build After Level 4)

  • BOB-051 Background Job Initiation - Foundation for all processing
  • BOB-052 Sequential Loan Processing - Core processing loop
  • BOB-053 PDF Naming Convention - Required before delivery
  • BOB-054/055/056 PDF Delivery Methods - Sequential fallback chain

Phase Alignment (MVP Strategy)

All 36 tickets are designated as PHASE 1 MVP. The three-tier approach ensures proper sequencing within the MVP phase.

MVP Delivery Approach

  • Sprint 1 (Tier 1): Complete all foundation tickets (BOB-001 to BOB-010). Establishes platform capabilities.
  • Sprint 2 (Tier 2): Complete all identification methods (BOB-021 to BOB-027). Enables loan population selection.
  • Sprint 3-4 (Tier 3): Complete grid display (BOB-031 to BOB-040). Users can view loan validation summaries.
  • Sprint 5-6 (Tier 4): Complete background jobs & PDF delivery (BOB-051 to BOB-060). Full bundle building capability.
  • Sprint 7 (Tier 5): Complete Activity Log features (BOB-071 to BOB-076). Job history & tracking.

MVP Success Criteria

  • Users can select loan populations via Manual Entry (minimum)
  • Users can view loan validation summaries in grid
  • Users can initiate bundle builds (Ready to Bundle OR Selected Loans)
  • System processes loans in background and delivers PDFs to destinations
  • Users can track job history via Activity Log

Risk Mitigation & Recommendations

  • BytePro API Dependency (BOB-007): This is a CRITICAL external dependency. Coordinate closely with BytePro team to ensure Byte 25.2 API availability. Consider building mock API endpoints for parallel development if API is delayed.
  • Database Schema Changes (BOB-008): Activity Log table must be deployed to all environments early. Include in initial database migration scripts to avoid blocking downstream features.
  • Trade/Filter/Commitment Schemas (BOB-022, BOB-023, BOB-024): These tickets have OPEN ISSUES regarding database schema. Resolve these unknowns during Tier 1 development to prevent delays in Tier 2.
  • Performance Testing (BOB-027, BOB-040): Maximum population size is unknown. Conduct load testing during Tier 2 to determine limits before building Tier 3 features.
  • Security Review (BOB-001): Three approved security roles are TBD. Complete security design review early to avoid rework in dependent tickets.
  • SFTP Credentials (BOB-055): Requires dbo.BundleSFTPCredentials table and secure credential storage. Plan encryption strategy during Tier 1.
